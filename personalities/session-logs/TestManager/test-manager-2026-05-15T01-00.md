## Key findings and observations

### Architecture gaps
- **No "add to shared plot" gesture (FEAT-002):** The only way to get a photo into a shared plot is via a trellis. The `addPlotItem` API exists but no UI surface calls it.
- **No web → Android device pairing:** A web-only user has no way to link a new Android device to their account. The pairing flow is Android → Web only.
- **No friends list on web (WEB-001):** `getFriends` exists in the web API client but there is no UI surface for it.

### Staging environment gaps
- GCS CORS missing for staging domain (fixed in session)
- No deployment runbook — commands had to be reconstructed from session logs (OPS-001 queued)
- Staging API key hardcoded in task files and git history (SEC-010 queued)
- Invite URLs on staging Android app point to production domain (BUG-008)

### Testing infrastructure
- TST-005 (Playwright infrastructure) is now unblocked — ARCH-002 and TST-003 both complete
- TST-006 queued to investigate Android E2E automation (Maestro/UI Automator/Appium)
- Manual testing revealed several bugs that automated tests would have caught earlier (BUG-003, BUG-004)

### Positive findings
- E2EE upload → friend share → receipt works end-to-end on web (BUG-007 validated)
- Web drag-drop, paste upload, and auto-approve trellis all work
- Shared plot invite, staging, and approval flow works end-to-end (with workarounds)
- Registration and vault unlock flow clean for new users
