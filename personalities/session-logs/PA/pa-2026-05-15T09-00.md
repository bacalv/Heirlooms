## Key decisions and findings

- **v0.54 held from production**: Too many gaps found during smoke test to justify release. Production stays at v0.53.1 until BUG-020/022/WEB-001 resolved and another smoke test passes.

- **Environment naming corrected**: What Bret calls "test" is what the PA had been calling "staging". PA_NOTES updated. "Staging" is reserved for the future prod-snapshot environment (OPS-003).

- **Test environment is throw-away**: All test accounts (smokeA, smokeB, smoke-test-plot etc.) are disposable. Production data must be protected from test activity. GCS bucket currently shared — a known gap.

- **BUG-018 root cause**: Shared plot trellises with `requires_staging = false` auto-inserted items into `plot_items` without DEK re-wrapping. Members couldn't decrypt. Fix: force `requires_staging = true` for all shared-plot trellises + Flyway V31 migration + defensive guard in `runUnstagedTrellisesForUpload`.

- **BUG-016 fully resolved**: Server-side `ON CONFLICT DO NOTHING` fix confirmed working with a clean test account (smokeA). Old Fire2 account had corrupt key state — fixed by clearing device_id from DB.

- **E2EE tag scheme decided**: Tags to be stored as HMAC tokens (keyed to master key) — server can evaluate criteria by token equality but never reads tag values. Residual leakage (equality/frequency) accepted and to be disclosed to users. See ARCH-007, SEC-012.

- **Smoke test defined**: 6-step shared plot E2E smoke test documented in TST-008. Steps 1–4 partially run; Steps 5–6 defined but not yet executed. Steps 4–5 blocked by BUG-020/022.

- **Device revocation gap (SEC-011)**: No in-app way to revoke an old device before selling/disposal. Android Keystore wipe on factory reset provides primary protection; server-side revocation is the missing safety net.

- **DB connection pattern**: Cloud SQL Proxy + psql via URL-encoded password works. Proxy binary at `/opt/homebrew/share/google-cloud-sdk/bin/cloud-sql-proxy`, psql at `/opt/homebrew/Cellar/libpq/18.3/bin/psql`. Use Python URL-encode for special chars in password.
