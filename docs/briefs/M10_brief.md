# Milestone 10 — Shared Plots

Four increments. Server + Web throughout; Android in E4.

Target version: v0.47.0 (E1) through v0.50.0 (E4).

---

## Context and scope

M10 delivers two related things: a richer filter/predicate system that replaces the
current tag-only plot criteria, and shared plots — named collections visible to multiple
users. The predicate system is the foundation; shared plots are built on top of it.

**In scope:**
- Predicate/criteria expression language replacing `plot_tag_criteria`
- Plot visibility (`private` / `shared` / `public`) on a unified `plots` table
- Hidden plots (`show_in_garden = false`) as reusable predicate building blocks
- Flows: automatic routing from a criteria expression into any collection plot
- Staging areas: per-flow review gate before items enter a collection plot
- Shared plots: multi-user collection plots with a per-plot group key (E2EE)
- Invitation: friends list first, invite link as fallback

**Out of scope:**
- Public plots (always require staging — deferred; `public` visibility reserved in schema)
- Plot key rotation on member removal (additive-only membership in M10)
- Real-time push for staging updates (client polls)
- Plot reordering per member (sort_order is plot-level in M10)
- Unfriend / member removal UI (schema supports it; no UI in M10)

---

## Design decisions (confirmed)

| Decision | Choice |
|---|---|
| Criteria format | JSONB expression tree on `plots.criteria`; replaces `plot_tag_criteria` junction table |
| Hidden predicates | Plots with `show_in_garden = false`; same table, same criteria format |
| Plot visibility | `private` (default), `shared`, `public` (future) on unified `plots` table |
| Collection plots | `criteria IS NULL`; contents stored in `plot_items` table |
| Query plots | `criteria IS NOT NULL`; contents derived by evaluating expression |
| Flow target | Any collection plot (`criteria IS NULL`), any visibility |
| Staging flag | Per-flow `requires_staging BOOLEAN`; public plots enforce `true` server-side |
| Staging decisions | Per `(plot_id, upload_id)`; scoped to the plot, not the flow |
| Rejected items | Revisable — delete decision row to un-reject; item reappears in staging |
| Shared plot E2EE | Per-plot group key (AES-256-GCM); plot key wrapped to each member's sharing pubkey |
| Member roles | `owner` (add, remove items, invite, delete plot) / `member` (add items, invite) |
| Invitation | Friends list picker (primary); invite link/token (fallback for non-friends) |
| `plot_ref` safety | Circular references detected at evaluation time; cross-user refs disallowed |

## Predicate/criteria expression language

The `criteria` column is a JSON expression tree. The server evaluates it into a
parameterised SQL WHERE clause — no PostgreSQL JSON operators used on the column itself.

**Atom types:**

```json
{"type": "tag",           "tag": "Sadaar"}
{"type": "media_type",    "value": "image"}          // or "video"
{"type": "taken_after",   "date": "2024-01-01"}
{"type": "taken_before",  "date": "2024-12-31"}
{"type": "uploaded_after","date": "2024-01-01"}
{"type": "uploaded_before","date": "2024-12-31"}
{"type": "has_location"}
{"type": "near",          "lat": 51.507, "lng": -0.127, "radius_metres": 500}
{"type": "device_make",   "value": "Apple"}
{"type": "device_model",  "value": "SM-T517D"}
{"type": "is_received"}
{"type": "received_from", "user_id": "uuid"}
{"type": "in_capsule"}
{"type": "just_arrived"}
{"type": "composted"}
{"type": "plot_ref",      "plot_id": "uuid"}         // inline another plot's criteria
```

**Composition:**

```json
{"type": "and", "operands": [...]}
{"type": "or",  "operands": [...]}
{"type": "not", "operand":  {...}}
```

**Example — "Pictures of Sadaar, not Bret":**

```json
{
  "type": "and",
  "operands": [
    {"type": "media_type", "value": "image"},
    {"type": "plot_ref", "plot_id": "uuid-of-tagged-sadaar-predicate"},
    {"type": "not", "operand": {"type": "tag", "tag": "Bret"}}
  ]
}
```

**`near` and E2EE:** GPS coordinates are stored in plaintext for legacy uploads
only. For M7+ encrypted uploads, `latitude`/`longitude` are NULL in the database
(stored in `encrypted_metadata`). The `near` predicate therefore applies to
pre-M7 items only in M10. A future increment can add a coarse plaintext geohash
as an opt-in for encrypted items.

**`plot_ref` evaluation:** the server resolves references recursively, tracking
visited plot IDs to detect cycles (fail loudly). A plot may only reference its
own user's plots. A shared plot's criteria referencing a private plot is permitted
(the query runs as the plot owner), but the result is visible to all members —
the SE should surface a warning in the creation UI.

## E2EE — Per-plot group key

Each shared plot has a symmetric AES-256-GCM `plot_key` generated by the owner at
creation time.

**Key storage:**
- `plot_members.wrapped_plot_key`: `plot_key` wrapped to the member's account sharing
  pubkey using `p256-ecdh-hkdf-aes256gcm-v1` (same algorithm as M9 item sharing).
- Each member (including the owner) has their own wrapped copy.
- New members: wrap current `plot_key` to their sharing pubkey, insert `plot_members` row.

**Item wrapping:**
- When an item is added to a shared plot, the client:
  1. Unwraps the item's DEK (using master key for own items; sharing privkey for received items).
  2. Fetches own `wrapped_plot_key` from `GET /api/plots/:id/plot-key`; unwraps with sharing privkey.
  3. Wraps item DEK and thumbnail DEK under `plot_key` using AES-256-GCM.
  4. POSTs to `POST /api/plots/:id/items` with the re-wrapped DEKs.
- `plot_items.wrapped_item_dek` format: new envelope type `plot-aes256gcm-v1`.

**Item decryption (member reading):**
1. Fetch `wrapped_plot_key` from `GET /api/plots/:id/plot-key`.
2. Unwrap with own sharing privkey → `plot_key`.
3. Unwrap `plot_items.wrapped_item_dek` with `plot_key` → item DEK.
4. Decrypt content with item DEK.

**Key rotation:** deferred. Members cannot be removed in M10 (schema supports it;
no UI). Plot key rotation on member removal is an M11+ concern.

## Schema changes

### V24 — Predicate/criteria system

```sql
-- V24__predicate_criteria.sql

ALTER TABLE plots
    ADD COLUMN criteria       JSONB   NULL,
    ADD COLUMN show_in_garden BOOLEAN NOT NULL DEFAULT TRUE,
    ADD COLUMN visibility     TEXT    NOT NULL DEFAULT 'private'
        CHECK (visibility IN ('private', 'shared', 'public'));

-- System plot gets a first-class criteria expression
UPDATE plots
    SET criteria = '{"type": "just_arrived"}'::jsonb
    WHERE is_system_defined = TRUE AND name = '__just_arrived__';

DROP TABLE plot_tag_criteria;

CREATE INDEX idx_plots_garden
    ON plots(owner_user_id, show_in_garden, sort_order)
    WHERE show_in_garden = TRUE;
```

### V25 — Shared plots, flows, staging

```sql
-- V25__shared_plots_flows_staging.sql

-- Members of shared/public plots (including the owner with role='owner')
CREATE TABLE plot_members (
    plot_id           UUID        NOT NULL REFERENCES plots(id) ON DELETE CASCADE,
    user_id           UUID        NOT NULL REFERENCES users(id),
    role              TEXT        NOT NULL CHECK (role IN ('owner', 'member')),
    wrapped_plot_key  BYTEA       NULL,    -- plot_key wrapped to this member's sharing pubkey
    plot_key_format   TEXT        NULL,    -- 'p256-ecdh-hkdf-aes256gcm-v1'
    joined_at         TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (plot_id, user_id)
);

-- Items explicitly added to a collection plot (criteria IS NULL)
CREATE TABLE plot_items (
    id                   UUID        PRIMARY KEY,
    plot_id              UUID        NOT NULL REFERENCES plots(id) ON DELETE CASCADE,
    upload_id            UUID        NOT NULL REFERENCES uploads(id),
    added_by             UUID        NOT NULL REFERENCES users(id),
    source_flow_id       UUID        NULL,
    wrapped_item_dek     BYTEA       NULL,    -- item DEK wrapped under plot_key
    item_dek_format      TEXT        NULL,    -- 'plot-aes256gcm-v1'
    wrapped_thumbnail_dek BYTEA      NULL,
    thumbnail_dek_format  TEXT       NULL,
    added_at             TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (plot_id, upload_id)
);

-- Flows: criteria expression → target collection plot
CREATE TABLE flows (
    id               UUID        PRIMARY KEY,
    user_id          UUID        NOT NULL REFERENCES users(id),
    name             TEXT        NOT NULL,
    criteria         JSONB       NOT NULL,
    target_plot_id   UUID        NOT NULL REFERENCES plots(id),
    requires_staging BOOLEAN     NOT NULL DEFAULT TRUE,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

ALTER TABLE plot_items
    ADD CONSTRAINT fk_source_flow FOREIGN KEY (source_flow_id) REFERENCES flows(id);

-- Per-item staging decisions (scoped to the plot, not the flow)
-- No row = pending (appears in staging if criteria still matches and requires_staging = true)
CREATE TABLE plot_staging_decisions (
    plot_id        UUID        NOT NULL REFERENCES plots(id) ON DELETE CASCADE,
    upload_id      UUID        NOT NULL REFERENCES uploads(id),
    decision       TEXT        NOT NULL CHECK (decision IN ('approved', 'rejected')),
    source_flow_id UUID        NULL REFERENCES flows(id),
    decided_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (plot_id, upload_id)
);

CREATE INDEX idx_plot_staging ON plot_staging_decisions(plot_id, decision);

-- Invite tokens for shared plot membership (fallback to friends list)
CREATE TABLE plot_invites (
    id             UUID        PRIMARY KEY,
    plot_id        UUID        NOT NULL REFERENCES plots(id) ON DELETE CASCADE,
    created_by     UUID        NOT NULL REFERENCES users(id),
    token          TEXT        NOT NULL UNIQUE,
    used_by        UUID        NULL REFERENCES users(id),
    used_at        TIMESTAMPTZ NULL,
    expires_at     TIMESTAMPTZ NOT NULL,
    created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

## Staging — derived query (not stored)

Staging for a plot is computed at request time:

```sql
-- Items pending review for plot :plot_id
SELECT DISTINCT u.*
FROM uploads u
JOIN flows f ON f.target_plot_id = :plot_id
           AND f.requires_staging = TRUE
           AND f.user_id = u.user_id        -- only route own items
WHERE [f.criteria evaluates to TRUE for u]
  AND NOT EXISTS (
      SELECT 1 FROM plot_staging_decisions psd
      WHERE psd.plot_id = :plot_id AND psd.upload_id = u.id
  )
  AND NOT EXISTS (
      SELECT 1 FROM plot_items pi
      WHERE pi.plot_id = :plot_id AND pi.upload_id = u.id
  );
```

Un-rejecting: `DELETE FROM plot_staging_decisions WHERE plot_id = ? AND upload_id = ?`.
Item reappears in staging immediately.

## Server changes

### Updated plot endpoints (E1)

Existing plot CRUD updated to use `criteria` JSONB instead of `tag_criteria` array.
`GET /api/plots` response adds `show_in_garden`, `visibility`, `criteria` fields;
removes `tag_criteria`.

### New endpoints (E2 — flows + staging)

| Method | Path | Purpose |
|---|---|---|
| `GET` | `/api/flows` | List own flows |
| `POST` | `/api/flows` | Create flow (`criteria`, `target_plot_id`, `requires_staging`) |
| `PUT` | `/api/flows/:id` | Update flow |
| `DELETE` | `/api/flows/:id` | Delete flow |
| `GET` | `/api/flows/:id/staging` | Get staging items (derived query) |
| `POST` | `/api/plots/:id/staging/:uploadId/approve` | Approve: creates `plot_items` row |
| `POST` | `/api/plots/:id/staging/:uploadId/reject` | Reject: creates `plot_staging_decisions(rejected)` |
| `DELETE` | `/api/plots/:id/staging/:uploadId/decision` | Un-reject: removes decision row |
| `GET` | `/api/plots/:id/staging/rejected` | List rejected items for recovery |

### New endpoints (E3 — shared plots)

| Method | Path | Purpose |
|---|---|---|
| `GET` | `/api/plots/:id/plot-key` | Fetch own wrapped plot key |
| `GET` | `/api/plots/:id/members` | List members |
| `POST` | `/api/plots/:id/members` | Invite a friend (body: `{userId}`) |
| `POST` | `/api/plots/:id/invites` | Generate invite link/token |
| `POST` | `/api/plots/join` | Redeem invite token (adds to `plot_members`) |
| `GET` | `/api/plots/:id/items` | List items in a collection plot |
| `POST` | `/api/plots/:id/items` | Add an item manually (body: wrapped DEKs) |
| `DELETE` | `/api/plots/:id/items/:uploadId` | Remove an item (owner, or member's own) |

## Increment plan

### E1 — Predicate/criteria system (server + web)

**Server:**
- V24 migration.
- New `CriteriaEvaluator` that walks the expression tree and builds a parameterised
  SQL WHERE clause. Handles all atom types; detects `plot_ref` cycles.
- Updated `PlotHandler`: CRUD accepts/returns `criteria` JSONB; drops `tag_criteria`.
- Updated list-uploads path: plot-filtered queries now go through `CriteriaEvaluator`
  instead of the tag-join path.
- `__just_arrived__` system plot now evaluated via the `just_arrived` atom (same logic,
  different path).

**Web:**
- Plot creation/editing: replace tag list with criteria builder UI.
- Explore filter → "Save as plot" now captures the full filter state as a criteria
  expression (date, media type, location, tags), not just tags.
- Hidden plots (`show_in_garden = false`) are invisible in the Garden sidebar but
  available as `plot_ref` targets in the criteria builder.

### E2 — Flows + staging (server + web)

**Server:**
- V25 partial: `flows` + `plot_staging_decisions` tables.
- Flow CRUD endpoints.
- Staging derived query endpoint.
- Approve/reject/un-reject endpoints.

**Web:**
- Flow creation UI: criteria builder + target plot picker + staging toggle.
- Staging review UI per plot: pending items grid, approve/reject actions.
- Rejected items panel (collapsed by default, expandable for recovery).

### E3 — Shared plots + E2EE (server + web)

**Server:**
- V25 full: `plot_members`, `plot_items`, `plot_invites` tables.
- Plot key generation and distribution.
- Invitation endpoints (friends list + invite token).
- `plot_items` CRUD with DEK re-wrapping validation.
- Access control: `plot_members` check on all shared plot reads.

**Web:**
- Shared plot creation (sets `visibility = 'shared'`, generates `plot_key`,
  inserts owner into `plot_members`).
- Invite member UI: friend picker + invite link copy button.
- Shared plot view: renders `plot_items` collection, decrypts via plot key.
- Manual "Add to shared plot" action from photo detail.
- Shared plots appear in Garden sidebar (owner and members both see them).

### E4 — Android adoption

- Criteria builder (predicate expression → API payload).
- Flow creation and management.
- Staging review screen.
- Shared plot: creation, invitation, member view, item add.
- Plot key management in `VaultSession` (alongside `sharingPrivateKey`).
- `HeirloomsImage` / `EncryptedThumbnail`: new `plot-aes256gcm-v1` unwrap path.
